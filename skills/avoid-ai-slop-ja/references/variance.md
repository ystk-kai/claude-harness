# Variance (ムラ) — keep 5 kinds on purpose

AI slop is uniform. The fix is deliberate unevenness — but **only within the document type's budget** (see document-types.md). Variation is a tool to break the metronome, never decoration for its own sake.

## The 5 kinds to keep

1. **文長のムラ** — mix 1-line paragraphs with longer ones. Don't normalize every paragraph to the same length.
2. **密度のムラ** — pack names/numbers where it matters; stay light elsewhere. Not every sentence needs a statistic.
3. **トーンのムラ** — let a cool/blunt sentence break a neutral run (degree depends on doc type).
4. **結論のムラ** — a paragraph may end on "結局どちらとも言い切れない" when that is the honest state; not every paragraph must "land".
5. **詳細度のムラ** — be detailed where it counts, terse elsewhere. An even gloss everywhere reads generated.

## Also vary

- **文末** — don't end every sentence with 〜です／〜である／〜できる. Mix 体言止め, 断定, 短文, question where natural.
- **段落の閉じ方** — drop the reflexive "〜だと感じました/〜が重要です" closer.

## Stop conditions (don't over-apply)

- If you find yourself adding typos, snark, or "まあそこそこ" to *seem* human in a briefing/technical/legal doc → stop; that's the wrong tool for the type.
- If every paragraph now has a forced "blunt aside" → you've built a new template. Remove most.
- Variation should be invisible as technique. If a reader notices "this is trying to sound human", it failed.
