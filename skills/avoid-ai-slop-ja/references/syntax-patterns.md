# Syntax patterns — detect & fix

These catch "generated-ness" deeper than vocabulary. Make each a review question.

## 1. False agency（無生物主語）

Inanimate/abstract subjects performing judgment or intent.
- ✗「データが示している」→ ✓「(自分が)見るとこうなっている」/「ログを数えると N 件」
- ✗「課題が浮き彫りになる」→ ✓「実行して気づいた問題」
- ✗「文化が醸成される」→ ✓「誰が何をした結果そうなった」
- 大きな主語も同類: ✗「多くのエンジニアが」→ ✓「自分も含めて〜してしまう」
- 注意: 組織・人を主語にした「Anthropic が示した／Mozilla が報告した」は false agency ではない（主体が実在）。

## 2. 命題型 H2（断定見出しの押し付け）

Each heading carries a thesis → reads like a string of tiny TED talks.
- ✗「○○もまた同じ方向を指している」→ ✓「○○の話」/ 名詞句「○○の分界」
- Pile-up を避ける: 断定見出しが連続したら半分を機能的見出し（名詞句）へ。substantive な要点見出しは可だが連発しない。

## 3. 偽の対句（Not X. It's Y. / 「XではなくY」）

- ✗「単なる X ではなく、Y だ」→ ✓「Y だ」(Y を直接述べ、否定を削る)
- ✗「Aでない。Bでもない。Cだ」→ ✓「Cだ」
- 例外: 「件数ではなく修正時間で測る」のように Y が実質的な選択を示すなら可。中身のない反転だけを削る。

## 4. フック→反転の出だし

- ✗「ChatGPTで誰でも記事が書ける。しかし——」→ ✓ 控えめに始める「最近、手で書く方が早い気がする」
- 煽り出し→逆張りのテンプレを避ける。

## 5. メタ構造宣言

- ✗「以下 3 つの観点から説明します」→ 削除して本題を直接書く。

## 6. ❌/✅ 比較（スライド調 bad-then-good）

- ✗ bad例→good例の対カードを並べる → ✓ 地の文で展開。(図解でも「言い換え」だけの対比は避け、関係/閾値/境界を足す)

## 7. 3 項目並列の多用

- 「3つのポイント」を機械的に作らない。2 つか 1 つで足りるなら減らす。並列が 3 回連続したら別形式へ。

## 8. 文末の単調

- 〜です／〜ます／〜である／〜できる/〜でしょう の連続を割る。断定・体言止め・短文・問いを必要箇所に混ぜる。

## 9. 一文圧縮（Wikipedia 調）

- 年代・人名・評価を 1 文に詰め込まない。確度の低い情報は伝聞で「およそ N 日で設計されたらしい」。

## 10. 儀式化した保険・遠い話者

- 各節末の「個人の見解ですが」を 1 記事 1〜2 回に。
- ✗「現代社会では重要だ」→ ✓「最近気になる」/「この環境では効く」。
